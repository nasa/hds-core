# HDS Core

**HDS Core** (`@nasa/hds-core`) is a Sass/CSS theme layer that configures the [U.S. Web Design System (USWDS)](https://designsystem.digital.gov/) with NASA's **Horizon Design System (HDS)** values — colors, typography, spacing, icons, and component styles.

HDS is NASA's unified visual language for digital products. USWDS is the open-source design system used across U.S. federal government websites. HDS Core bridges the two: all standard USWDS components and utility classes continue to work, themed to match HDS.

## What's included

HDS Core v1.0 provides:

- **Foundations** — Color, palettes, typography, spacing, grid, and icons
- **Components** — Buttons, links, pagination, breadcrumbs, accordion, forms, tables, and more
- **Palette system** — Six color palettes that automatically adapt every component

This is a CSS-only package — it does not add any JavaScript. HDS Core does assume that your project includes the standard [USWDS JavaScript](https://designsystem.digital.gov/documentation/getting-started/developers/phase-two-compile/#step-4-add-the-uswds-javascript) for interactive components like accordions and banners. Components that require JavaScript beyond what USWDS provides (modals, date pickers, carousels) are planned for future releases. See the [Roadmap](?path=/docs/roadmap--docs) for details.

## Key concepts

**Red means "go somewhere," blue means "do something here."** HDS uses NASA Red for actions that navigate to a new page (`.usa-button`) and NASA Blue for on-page interactions (`.usa-button--secondary`, `.usa-button--outline`). This wayfinding rule applies consistently across all components.

**Palettes are automatic.** Wrap any section in a palette class and every HDS component inside it adapts — text, borders, backgrounds, buttons, links. You don't need to set individual colors.

```html
<div class="hds-palette-dark">
  <!-- Everything inside adapts automatically -->
</div>
```

## Using this Storybook

- **Foundations** document the design tokens and systems (color, type, spacing)
- **Components** each have a **Guidance** tab (usage docs) and a **Playground** tab (interactive example)
- Click **"Show code"** on any example to copy the markup
- Use the **palette switcher** (paintbrush icon in the toolbar) to preview any component on all six palettes
- Use the **viewport switcher** (phone icon in the toolbar) to test components at different screen sizes — useful for verifying responsive behavior on Playground tabs
- Use the **Measure tool** (ruler icon in the toolbar) to inspect pixel spacing between elements

## Installation

```bash
npm install @nasa/hds-core @uswds/uswds
```

See the [README](https://github.com/nasa/hds-core) for full setup instructions including Sass load paths and pre-compiled CSS options.

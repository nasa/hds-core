# Button

A button draws attention to important actions with a large selectable
surface. HDS buttons use USWDS `.usa-button` classes with NASA brand
colors — no markup changes needed for existing USWDS sites
(**Tier 1 override**).

## Primary arrow button

`.hds-btn--primary` is an **HDS-only component (Tier 3)** — a red
pill-shaped link with an animated CSS arrow. Use it for editorial CTA
links in hero banners and content cards.

<p className="hds-caption">
  Use the palette switcher (🖌 toolbar) to preview on all six HDS backgrounds.

- Always an `<a>` tag — it's a navigation link, never a form action
- Pairs with `.usa-link--external` for external destinations — the
  arrow changes from a line to a diagonal
- Palette-aware — text and arrow adapt automatically on dark backgrounds

<Note type="code">
The arrow is a pure CSS `::after` pseudo-element using a data-URI SVG,
not an icon from the sprite. It inherits color automatically and works
without the sprite sheet loaded.
</Note>

<Note type="figma">
Figma shows 6 size variants (14px–36px). HDS Core currently ships the
default size only. Additional sizes are pending creative director review.
</Note>

## Outline button

A lower-emphasis on-page action with a NASA Blue border. On dark
backgrounds, the border stays NASA Blue and only the text flips to white.

Two mechanisms provide dark-background styling:

- **Palette-aware (automatic):** `.usa-button--outline` inside an HDS
  palette wrapper adapts automatically. No extra class needed.
- **Manual fallback:** Add `.usa-button--inverse` for dark backgrounds
  without a palette wrapper.

USWDS `--inverse` turns the entire outline button white (border + text).
In HDS Core, only the text flips to white — the border stays NASA Blue.
</Note>

## When to use the button component

- Use the **primary arrow** (`.hds-btn--primary`) for editorial CTA
  links in hero sections and content cards.
- Use the **Call to Action** (red, `.usa-button`) when the action
  **navigates the user away** — downloads, applications, key tasks.
- Use the **Secondary Filled** (blue, `.usa-button--secondary`) when
  the action **stays on the current page** — submit, filter, search.
- Use the **Outline** (`.usa-button--outline`) for lower-emphasis
  on-page actions.
- Use a button for **important actions** you want users to take on
  the page.

## When to consider something else

- If the action is **less important**, consider a text link instead.
- For **icon-only actions** (play, download, share), use
  [Icon Button](?path=/story/components-icon-button--all-roles) instead.
- For **inline text navigation** within body content, use
  [Link](?path=/story/components-link--default).
- If a **navigation link looks like a button**, use an `<a>` tag with
  button classes — not a `<button>`. Correct semantics matter for
  assistive technology.
- Avoid more than **one red button per visible section**. Pair a CTA
  with secondary or outline buttons to establish hierarchy.

## Usability guidance

- **Be concise.** Use action verbs: "Download", "Submit", "Apply Now."
  Avoid vague labels like "Click here" or "Go."
- **Labels should make sense out of context.** Screen reader users may
  encounter buttons without surrounding text.
- **Use sentence case** for button labels.

<Note type="figma">
The HDS Figma spec does not define a button active state. HDS Core
uses the same visual treatment for hover and active, consistent with
Apple HIG.
</Note>

## Accessibility

- Use **semantic elements:** `<button>` for actions, `<a>` for
  navigation. If a link looks like a button, apply button classes
  to the `<a>`.
- Use **`disabled`** for truly inactive buttons — removed from tab
  order and announced as disabled by screen readers.
- Use **`aria-disabled="true"`** when the button should remain
  **focusable** — for example, to explain via tooltip why the action
  is unavailable. Both receive identical visual styling.
- **Don't remove focus styles.** HDS Core applies a visible focus ring
  on all interactive elements.
- **Write descriptive labels.** "Download fiscal year 2025 report"
  is better than "Download." Supplement short labels with `aria-label`
  or `aria-describedby` when needed.

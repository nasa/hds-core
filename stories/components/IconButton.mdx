# Icon Button

Circle-shaped buttons containing a single icon and no visible text. Icon buttons are an **HDS-only component (Tier 3)** — USWDS has no equivalent. Use them for compact actions where space is limited and the icon's meaning is universally understood.

Icon buttons follow the **HDS wayfinding color rule**: red navigates away, blue stays on the page, neutral colors are for UI controls.

## Roles

Every icon button requires a role class that determines its color. The role communicates the button's intent at a glance.

## Sizes

Three sizes are available. All share the same role classes.

| Size    | Class               | Use                               |
| ------- | ------------------- | --------------------------------- |
| Small   | `.hds-btn-icon--sm` | Inline with text, tight layouts   |
| Default | _(no modifier)_     | Standard standalone actions       |
| Large   | `.hds-btn-icon--lg` | Hero sections, prominent controls |

## When to use the icon button component

- **Compact actions** where the icon's meaning is universally clear — play, download, share, close, expand.
- **Social media rows** — a group of social/sharing icons.
- **Action bars** — a horizontal row of secondary actions below content.
- **CTA accent** — a small CTA icon paired with adjacent text (e.g., "Explore the mission" + arrow button).

## When to consider something else

- If the action needs a **text label** for clarity, use a [Button](?path=/docs/components-button--guidance) instead.
- For **inline text navigation**, use a [Link](?path=/docs/components-link--guidance).
- If the icon's meaning is **not universally obvious** (e.g., a domain-specific glyph), use a text button — don't rely on `aria-label` alone to convey meaning to sighted users.
- For **decorative icons** that aren't interactive, use `.hds-glyph` instead of an icon button.

## Usability guidance

- **Always provide `aria-label`.** Icon buttons have no visible text — the label is the only way screen readers and voice control users can identify the action.
- **Write action-oriented labels.** "Download file" is better than "Download." "Go to mission page" is better than "Arrow."
- **Use `<button>` for actions, `<a>` for navigation.** If the icon button navigates to a new page, use an `<a>` tag. If it triggers an on-page action (toggle, submit), use `<button>`.
- **One icon per button.** Don't combine multiple icons in a single icon button.

## Usage patterns

### Social row

Group social/sharing icons in a horizontal row using the `--social` role. Use `<a>` tags since these navigate to external platforms.

<Canvas of={IconButtonStories.SocialRow} />

### Action bar

Combine multiple on-page actions in a toolbar-style row. Mix roles to communicate hierarchy — secondary for the primary action, outline and utility for supporting actions.

<Canvas of={IconButtonStories.ActionBar} />

### CTA with adjacent text

Pair a small CTA icon button with adjacent text for a compact navigational call-to-action. This is distinct from the [primary arrow button](?path=/docs/components-button--guidance), which is a self-contained pill-shaped element.

## Hover and disabled states

Hover and disabled states for icon buttons are **not yet implemented**. This is tracked as pending work. Currently, icon buttons don't change appearance on hover and don't have a dedicated disabled visual treatment.

## Accessibility

- **`aria-label` is required** on every icon button. Without it, the button is announced as an empty button by screen readers.
- **`aria-hidden="true"` on the SVG.** The icon is decorative — the `aria-label` on the button provides the accessible name. Adding `focusable="false"` on the SVG prevents a double-focus issue in older versions of IE/Edge.
- **Don't remove focus styles.** HDS Core applies a visible focus ring on all interactive elements.
- **Touch target size.** Even the small size meets the 24×24px minimum recommended by WCAG 2.5.8 (Target Size). The default and large sizes exceed it comfortably.

```html
<!-- ✅ Correct -->
<button class="hds-btn-icon hds-btn-icon--secondary" type="button" aria-label="Download report">
  <svg class="hds-icon" aria-hidden="true" focusable="false">
    <use xlink:href="/assets/img/hds-sprite.svg#download"></use>
  </svg>
</button>

<!-- ❌ Missing aria-label -->
<button class="hds-btn-icon hds-btn-icon--secondary" type="button">
  <svg class="hds-icon" aria-hidden="true" focusable="false">
    <use xlink:href="/assets/img/hds-sprite.svg#download"></use>
  </svg>
</button>
```

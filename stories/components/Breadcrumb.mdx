# Breadcrumb

Breadcrumbs are a secondary navigation component that helps visitors understand where the current page is in the site hierarchy, and allows them to navigate back to higher-level pages. HDS breadcrumbs use USWDS `.usa-breadcrumb` markup with HDS theming (**Tier 1 override**).

## Depth variants

Breadcrumbs consist of a list of links, separated by forward slashes, beginning with the homepage and ending with the current page. The current page is always the last item — displayed in a distinct color but never as a link.

**Maximum 3 visible elements and 2 slashes**, regardless of how deep the page is.

## When to use the breadcrumb component

- **Multi-level site hierarchies** — any page that is 2+ levels deep in the site structure.
- **Below global navigation** — displayed on most pages at Large and XL screen sizes.
- **Wayfinding** — helping users orient themselves and navigate back up the hierarchy.

## When to consider something else

- **Single-level sites** — if there's no meaningful hierarchy, breadcrumbs add noise.
- **Step-by-step processes** — use a step indicator instead. Breadcrumbs show hierarchy, not progress.
- **Mobile navigation** — breadcrumbs are typically hidden on small screens. Ensure other navigation patterns (back button, menu) cover the same paths.

## Usability guidance

- **Always include the current page.** The last item in the breadcrumb shows the user where they are. It's displayed in a distinct color but is never a clickable link.
- **Start with Home** for 2- and 3-level breadcrumbs. The first item links to the site root.
- **Use ellipsis for 4+ levels.** Home disappears, and the ellipsis becomes the first element. Only 3 elements are ever visible.
- **Use real page names.** "Artemis I" is better than "Mission Detail."
- **Max 2 slashes.** If you see 3+ slashes, the breadcrumb is too deep — truncate with ellipsis.

## Light and dark palettes

Breadcrumbs adapt automatically inside palette containers:

| Element                   | Light palettes         | Dark palettes             |
| ------------------------- | ---------------------- | ------------------------- |
| Non-active text + slashes | Carbon 60 (muted)      | Carbon 40 (muted)         |
| Current page              | Carbon Black (heading) | Spacesuit White (heading) |
| Hover text                | Carbon Black (heading) | Spacesuit White (heading) |
| Hover weight              | Semibold               | Semibold                  |
| Hover underline           | Dotted                 | Dotted                    |

<Note type="figma">
  The Figma spec tested breadcrumbs on white, light, dark, and black backgrounds only. Midtone and
  blue palettes use the same palette variables and may need creative director review.
</Note>

## Accessibility

From the HDS Figma accessibility guidance:

- **Keyboard navigation:** Users trigger a breadcrumb link by pressing **ENTER/RETURN** while the link has focus. Users navigate between breadcrumb links by pressing **TAB** and **SHIFT-TAB**.
- **ARIA navigation landmark:** The breadcrumb component is contained within an ARIA `navigation` landmark region (`<nav>` with `aria-label="Breadcrumbs"`).
- **`aria-current="page"`:** The current page element must contain the `aria-current="page"` attribute.
- **Ordered list (`<ol>`):** Screen readers announce position and count (e.g., "item 2 of 3"), reinforcing the hierarchical structure.
- **Current page is not a link.** It's where the user already is — linking to it creates confusion, especially for screen reader users.

```html
<!-- ✅ 3 levels -->
<nav class="usa-breadcrumb" aria-label="Breadcrumbs">
  <ol class="usa-breadcrumb__list">
    <li class="usa-breadcrumb__list-item">
      <a class="usa-breadcrumb__link" href="/">Home</a>
    </li>
    <li class="usa-breadcrumb__list-item">
      <a class="usa-breadcrumb__link" href="/missions">Missions</a>
    </li>
    <li class="usa-breadcrumb__list-item usa-current" aria-current="page">
      <span>Artemis I</span>
    </li>
  </ol>
</nav>

<!-- ✅ 4+ levels (ellipsis, Home hidden) -->
<nav class="usa-breadcrumb" aria-label="Breadcrumbs">
  <ol class="usa-breadcrumb__list">
    <li class="usa-breadcrumb__list-item">
      <span class="usa-breadcrumb__link">…</span>
    </li>
    <li class="usa-breadcrumb__list-item">
      <a class="usa-breadcrumb__link" href="/missions/artemis">Artemis</a>
    </li>
    <li class="usa-breadcrumb__list-item usa-current" aria-current="page">
      <span>Artemis I Overview</span>
    </li>
  </ol>
</nav>
```

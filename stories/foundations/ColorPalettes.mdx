# Color Palettes

HDS provides six color palettes that automatically adapt typography, links, buttons, icons, and borders to their background. Wrap any section in a palette class and every HDS component inside it adjusts automatically.

For the full color reference, see [Color](?path=/docs/foundations-color--docs).

## How to apply a palette

Use a class or data attribute on any container element:

```html
<section class="hds-palette-dark">
  <!-- Everything inside adapts automatically -->
</section>

<section data-hds-palette="dark">
  <!-- Same result -->
</section>
```

**White is the default.** If no palette is assigned to a parent element, white palette coloring is used. You don't need to add `hds-palette-white` unless you're overriding a section inside a different palette.

## Using palette tokens in custom styles

Inside a palette wrapper, `--hds-palette-*` CSS custom properties automatically resolve to the correct colors for that palette. Use these instead of hardcoded color values so your custom styles adapt to any palette context:

```css
.my-component {
  color: var(--hds-palette-text);
  background: var(--hds-palette-bg);
  border-color: var(--hds-palette-border);
}

.my-heading {
  color: var(--hds-palette-heading);
}

.my-muted-text {
  color: var(--hds-palette-muted);
}

.my-link {
  color: var(--hds-palette-link-text);
  text-decoration-color: var(--hds-palette-link-underline);
}
```

These tokens cover text, headings, muted text (labels, metadata, captions), links, borders, backgrounds, button colors, and more. Every HDS component uses them internally — your custom styles can use the same tokens to stay consistent.

For the full list of available color values, see [Color](?path=/docs/foundations-color--docs).

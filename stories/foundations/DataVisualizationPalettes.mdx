# Data Visualization Palettes

Color palettes for use in charts and data visualizations. These palettes are **separate from the six HDS component palettes** — they are intended only for encoding data in charts, not for UI elements.

For general color guidance, see [Color](?path=/docs/foundations-color--docs). For chart design guidance, see [Data Visualization](?path=/docs/foundations-data-visualization--docs).

## How to use these palettes

HDS provides data visualization colors as CSS custom properties. Use them in CSS, Sass, or copy the hex values into your charting library config.

### In CSS or Sass

```css
.my-chart-series-1 {
  fill: var(--hds-dataviz-cat-1);
}
.my-chart-series-2 {
  fill: var(--hds-dataviz-cat-2);
}
.my-chart-series-3 {
  fill: var(--hds-dataviz-cat-3);
}
```

### In JavaScript charting libraries

Chart.js, Highcharts, Plotly, and ECharts accept hex strings. Copy the hex values from the palette tables below:

```js
const chart = new Chart(ctx, {
  data: {
    datasets: [
      {
        backgroundColor: ['#0b4778', '#fb5a47', '#fa9441'],
      },
    ],
  },
});
```

### Palette-aware behavior

On light backgrounds (white, light, midtone, blue), the properties resolve to light-background color values. Inside `.hds-palette-dark` or `.hds-palette-black`, they automatically switch to lighter values optimized for dark backgrounds. No extra classes or overrides needed — the same CSS works everywhere:

```html
<section class="hds-palette-dark">
  <!-- var(--hds-dataviz-cat-1) automatically resolves to the dark variant -->
  <svg>
    <rect fill="var(--hds-dataviz-cat-1)" />
  </svg>
</section>
```

### Choosing a palette type

**Categorical** — when comparing distinct categories (e.g., agencies, missions, planets). Colors are ordered to maximize contrast between adjacent items.

**Sequential** — when color represents a value or range (e.g., temperature, density, altitude). Use a single hue family from light to dark.

**Semantic** — when data categories have strong color associations (Mars = red, Earth = blue). Use the matching hue family to make the visualization more intuitive.

## Categorical palettes

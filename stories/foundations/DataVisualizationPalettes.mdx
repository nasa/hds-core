# Data Visualization Palettes

Color palettes for use in charts and data visualizations. These palettes are **separate from the six HDS component palettes** — they are intended only for encoding data in charts, not for UI elements.

For general color guidance, see [Color](?path=/docs/foundations-color--docs). For chart design guidance, see [Data Visualization](?path=/docs/foundations-data-visualization--docs).

## How to use these palettes

Use the USWDS `color()` function in Sass to reference any value:

```scss
.my-chart-series-1 {
  fill: color('blue-70v');
}
```

### Categorical data

When comparing distinct categories (e.g., agencies, missions, planets), use the categorical palettes below. Colors are ordered to maximize contrast between adjacent items.

### Sequential data

When color represents a value or range (e.g., temperature, density, altitude), use a single hue family from light (10) to dark (90).

### Semantic associations

When data categories have strong color associations (Mars = red, Earth = blue), use the matching hue family to make the visualization more intuitive.

<Note type="figma">The original HDS Figma spec defines data visualization colors using custom hex values. For HDS Core, these have been translated to the nearest USWDS color tokens for consistency with the USWDS utility system. Hex values shown below are the USWDS token values.</Note>

### Usage

```html
<!-- Class or data attribute вЂ” both work identically -->
<section class="hds-palette-dark">
<section data-hds-palette="dark">

<!-- No palette = normal USWDS behavior -->
<!-- USWDS utility classes (!important) override palette vars -->
```

### Flags

```scss
$hds-enable-experimental-palettes: false !default;  // Palettes 4-6
$hds-enable-auto-dark-mode: false !default;          // prefers-color-scheme
// Auto dark mode requires experimental flag. Both are permanently opt-in.
```

## Icon System

### Architecture

- **Glyph** (`.hds-icon`) вЂ” single-color SVG, always `currentColor`, always `aria-hidden="true"`
- **Container** (`.hds-btn-icon`) вЂ” CSS circle with palette-aware colors
- **Bare glyph** (`.hds-glyph`) вЂ” inline icon without container
- CSS references individual SVG files (mask-image can't use sprites)
- HTML uses sprite via `<svg><use href="hds-sprite.svg#icon-name"/></svg>`
- Both individual files and sprite are required in deployment

### SVG Refactoring Rules

**System/arrow/file/eye/interactive icons:**
- Remove `width`, `height` from `<svg>`
- Keep `viewBox="-2 -2 24 24"` (20px art centered in 24px frame)
- Remove background circles вЂ” CSS provides the container
- Change all `fill`/`stroke` values to `currentColor`

**Tag icons:**
- Keep built-in circle
- Circle fill в†’ `currentColor`
- Inner art stays `fill="white"` or `stroke="white"`

**Arrow/Circle-* icons:**
- Keep as-is with embedded circles (for devs who want self-contained SVGs)
- HDS Core code does not reference these вЂ” uses chevron glyphs instead

## Primary Arrow Button

- `.hds-btn--primary` вЂ” text + CSS `::after` red circle with white arrow
- Arrow direction auto-swaps: `в†’` internal, `в†—` external (via `.usa-link--external`)
- Data URI SVGs for arrow glyphs (hardcoded white per spec)
- `Primary-Arrow.svg` to be deleted

## External Links

- Uses USWDS `.usa-link--external` class (manual, explicit)
- HDS overrides USWDS launch icon в†’ HDS `arrow-line-diagonal.svg` via mask-image
- SR text provided by USWDS CSS (`::before` with sr-only content)
- Optional JS module (`hds-external-links.js`) вЂ” separate, not auto-loaded
- `.hds-link--internal` escape hatch for JS auto-detect

## Accessibility

- Icon buttons without text в†’ require `aria-label`
- Icons alongside text в†’ `aria-hidden="true" focusable="false"`
- `.hds-sr-only` utility class (uses `visually-hidden` mixin)
- Focus rings use `--hds-palette-link-text` with fallback to `color("primary")`
- Wayfinding rule enforced by component roles (red = navigate, blue = on-page)
- Screen reader testing with NVDA/VoiceOver needed before release

## Forms

- Light backgrounds only for now
- Dark palette form elements are TODO
- Current styles lean on USWDS defaults + HDS color patterns

## Print

- `@media print` in `_hds-print.scss`
- Forces all palettes to white background, black text
- Hides nav, footer, banner, icon buttons
- Shows link URLs after link text
- Prevents page breaks inside images and after headings

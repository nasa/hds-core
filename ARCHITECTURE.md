# HDS Core Architecture Decisions

**Last updated:** 2026-03-08

---

## Package

- **Name:** `@nasa/hds-core`
- **Foundation:** CMS-agnostic Sass package on top of `@uswds/uswds ^3.3.0`
- **Tech stack:** Vanilla JS (no jQuery), SCSS source with CSS output
- **Build tools:** Gulp + `@uswds/compile` for Sass, `gulp-svg-sprite` for icons
- **Status:** Draft/develop branch, no current consumers

## Sass Architecture

### Entry Point: `styles.scss`

```scss
@forward "hds-uswds-theme";    // 1. Configure USWDS with HDS values
@forward "uswds";              // 2. Full USWDS library
@forward "hds-custom-styles";  // 3. Foundations: tokens, mixins, base, print
@forward "hds-components";     // 4. USWDS overrides + HDS components
@forward "hds-palettes";       // 5. Palette system (opt-in per section)
```

### Load Order (Critical)

USWDS's Sass module system requires `uswds-core` to be configured with `@use "uswds-core" with (...)` before anything else loads it.

```
styles.scss
  → _hds-uswds-theme.scss
      @use "hds-tokens" as hds       ← Pure Sass, no uswds-core
      @use "uswds-core" with (...)   ← FIRST load, configured ✅
  → uswds                            ← Uses already-configured uswds-core ✅
  → _hds-custom-styles.scss          ← @use "uswds-core" as * (reuses) ✅
  → _hds-components.scss             ← @use "uswds-core" as * (reuses) ✅
  → _hds-palettes.scss               ← @use "hds-tokens" (pure Sass) ✅
```

⚠️ **`_hds-tokens.scss` CANNOT use `@use "uswds-core"`** — doing so would load uswds-core unconfigured before the theme file has a chance to configure it, causing a Sass compilation error.

### File Responsibilities

| File | Purpose |
|------|---------|
| `_hds-tokens.scss` | Pure Sass variables and maps. No USWDS dependency. Defines all HDS color hex values, type scale, weights, line heights, border settings. |
| `_hds-uswds-theme.scss` | Configures USWDS via `@use "uswds-core" with (...)`. Sets theme colors, typography, spacing, component settings. |
| `_hds-custom-styles.scss` | Foundations loaded after USWDS: CSS custom properties on `:root`, shared mixins, utility classes, base HTML element styles, print styles. |
| `_hds-components.scss` | USWDS component overrides (header, footer, buttons, forms, accordion, etc.) and HDS-specific components (icon buttons, primary arrow button). |
| `_hds-palettes.scss` | 6 color palette definitions with CSS custom properties. Stable (white, light, midtone) and experimental (dark, blue, black). |

## Color System

### The Challenge

USWDS theme settings only accept USWDS system token strings (e.g., `"gray-50"`), not arbitrary hex values. The HDS Carbon scale doesn't have exact matches in USWDS's system palette.

### The Solution

1. Theme settings use closest USWDS approximations (required by USWDS)
2. All HDS custom styles use exact `$hds-color-*` variables
3. USWDS components are overridden with exact HDS values in `_hds-components.scss`

### USWDS Theme Approximations

| HDS Color | Hex | USWDS Token | USWDS Hex | Delta |
|-----------|-----|-------------|-----------|-------|
| Spacesuit White | #FFFFFF | `"white"` | #FFFFFF | Exact ✅ |
| Carbon 05 | #F6F6F6 | `"gray-5"` | #f0f0f0 | Small |
| Carbon 10 | #E3E3E3 | `"gray-10"` | #e6e6e6 | Small |
| Carbon 50 | #77777A | `"gray-50"` | #757575 | Tiny |
| Carbon 80 | #2E2E32 | `"gray-80"` | #2d2d2d | Tiny |
| Carbon 90 | #17171B | `"gray-90"` | #1b1b1b | Tiny |
| Carbon Black | #000000 | `"black"` | #000000 | Exact ✅ |

### Color Convention in Code

| Context | Use | Example |
|---------|-----|---------|
| HDS brand/Carbon colors | `$hds-color-*` | `$hds-color-carbon-60` |
| USWDS state colors | `color()` | `color("error")` |
| Typography functions | USWDS functions | `family("heading")`, `size("body", "xs")` |
| Spacing | USWDS functions | `units(3)` |
| CSS/JS consumers | Custom properties | `var(--hds-color-carbon-60)` |

### Why Not `$global-color-palettes`?

USWDS 3.13.0's `$global-color-palettes` only accepts palette registry string keys (e.g., `"palette-color-default"`), not raw Sass maps. Attempting to pass a custom map causes compilation errors. HDS-specific colors are instead available via:
- `$hds-color-*` Sass variables (in Sass files)
- `var(--hds-color-*)` CSS custom properties (in CSS/JS)

## Palette System

### 6 Palettes

| # | Name | Background | Status |
|---|------|------------|--------|
| 1 | White | Spacesuit White #FFFFFF | Stable (default) |
| 2 | Light | Carbon 05 #F6F6F6 | Stable |
| 3 | Midtone | Carbon 20 #D1D1D1 | Stable |
| 4 | Dark | Carbon 90 #17171B | Experimental |
| 5 | Blue | NASA Blue Shade #0B3D91 | Experimental |
| 6 | Black | Carbon Black #000000 | Experimental |

### Usage

```html
<section data-hds-palette="white">
  <!-- Content uses white palette colors -->
</section>

<section data-hds-palette="dark">
  <!-- Content uses dark palette colors (requires experimental flag) -->
</section>
```

### Configuration Flags

```scss
// In your project's Sass, before importing HDS Core:
$hds-enable-experimental-palettes: true;  // Enable dark/blue/black
$hds-enable-auto-dark-mode: true;         // OS dark mode (requires experimental)
```

### CSS Custom Properties Per Palette

**Universal (same across all palettes):**
- `--hds-palette-btn-primary-bg` — NASA Red
- `--hds-palette-btn-filled-text` — Spacesuit White
- `--hds-palette-checkbox-fill` — NASA Blue
- `--hds-palette-social-fill` — Carbon 60
- `--hds-palette-social-icon` — Carbon 05

**Per-palette (varies by palette):**
- `--hds-palette-bg`
- `--hds-palette-heading`
- `--hds-palette-text`
- `--hds-palette-link-text`
- `--hds-palette-link-underline`
- `--hds-palette-link-arrow`
- `--hds-palette-muted`
- `--hds-palette-border`
- `--hds-palette-btn-secondary-bg`
- `--hds-palette-checkbox-text`
- `--hds-palette-checkbox-stroke`
- `--hds-palette-utility-fill`
- `--hds-palette-utility-stroke`
- `--hds-palette-utility-icon`
- `--hds-palette-icon`

### Override Behavior

- USWDS utility classes (`!important`) override palette vars
- No palette attribute = normal USWDS behavior

## Icon System (Themeable)

Icons in `hds-icons/` are single-color, themeable SVGs that compile into a sprite.

### Source Requirements

- **Location:** `src/img/hds-icons/`
- **Naming:** Lowercase-hyphenated with category prefix: `arrow-chevron-right.svg`, `file-archive.svg`, `tag-video.svg`
- **Fill:** Must use `fill="currentColor"` (not hardcoded colors)
- **ViewBox:** `viewBox="-2 -2 24 24"` (20px art centered in 24px frame)

### Icon Categories

| Category | Fill | Circle |
|----------|------|--------|
| System, arrow, file, eye, interactive | `currentColor` | Stripped (CSS provides container) |
| Tag icons | `currentColor` on circle | Keep built-in circle, white inner art |

### Build Process

1. Source SVGs in `src/img/hds-icons/`
2. `gulp-svg-sprite` generates `dist/assets/img/hds-sprite.svg`
3. IDs derived from filenames: `download.svg` → `#download`

### Usage in HTML

```html
<svg class="hds-icon" aria-hidden="true" focusable="false">
  <use xlink:href="/assets/img/hds-sprite.svg#download"></use>
</svg>
```

### Usage in CSS

CSS `mask-image` references individual SVG files (sprites don't work with mask-image):

```scss
mask-image: url('../assets/img/hds-icons/arrow-line-diagonal.svg');
```

## Button Graphics (Fixed-Color)

Graphics in `hds-buttons/` are fixed-color SVGs with colors baked in. They do NOT respond to palette theming and are NOT included in the sprite.

### Contents

| File | Description |
|------|-------------|
| `interactive-close.svg` | Interactives button: close/X |
| `interactive-less-info.svg` | Interactives button: collapse/minus |
| `interactive-location.svg` | Interactives button: map pin |
| `interactive-menu.svg` | Interactives button: hamburger menu |
| `interactive-more-info.svg` | Interactives button: expand/plus |
| `interactive-play.svg` | Interactives button: play |
| `primary-arrow.svg` | Red circle + white arrow (fallback for CSS implementation) |

### Interactives Appearance

- White outline circle
- NASA Blue fill
- White icon glyph
- Used for tooltip/popover triggers on interactive modules

### Usage

Reference as standalone files:

```html
<img src="/assets/img/hds-buttons/interactive-play.svg" alt="Play">
```

```scss
background-image: url('../assets/img/hds-buttons/primary-arrow.svg');
```

### Why Separate?

- These violate the icon system's `currentColor` contract
- Fixed colors mean they can't adapt to palettes
- Keeping them separate prevents developer confusion
- Original HDS classifies Interactives as Buttons (Elements), not Icons (Foundations)

## Icon Buttons

### Classes

| Class | Purpose |
|-------|---------|
| `.hds-btn-icon` | Interactive circle container (needs `aria-label` when standalone) |
| `.hds-icon` | SVG glyph inside button (always `aria-hidden="true" focusable="false"`) |
| `.hds-glyph` | Bare inline icon without container |

### Roles

| Modifier | Purpose | Colors |
|----------|---------|--------|
| `--cta` | Navigate to new page | NASA Red fill, white icon |
| `--secondary` | On-page actions (download, share) | NASA Blue fill, white icon |
| `--outline` | Lower emphasis on-page | NASA Blue border, dark icon |
| `--utility` | Carousel/accordion controls | Neutral/adaptive |
| `--social` | Social media links | Gray fill |
| `--interactive` | Small tooltip triggers | Neutral, smaller size |

### Sizes

| Modifier | Purpose |
|----------|---------|
| `--sm` | Inline with text |
| `--lg` | Hero CTAs |

## Primary Arrow Button

`.hds-btn--primary` generates text + CSS `::after` red circle with white arrow.

- Arrow direction auto-swaps: right (internal) → diagonal (external via `.usa-link--external`)
- Uses data URI SVGs for arrows (hardcoded white is correct per HDS spec)
- No SVG markup needed in HTML

```html
<a href="/page" class="hds-btn--primary">Learn More</a>
<a href="https://external.com" class="hds-btn--primary usa-link--external">External</a>
```

---

## External Links

- Uses USWDS's `.usa-link--external` class (must be added manually)
- HDS overrides USWDS launch icon → `arrow-line-diagonal.svg` via `mask-image`
- Screen reader text provided by USWDS CSS (`::before` with sr-only)
- `.hds-link--internal` escape hatch available for links that look external but aren't

## Accessibility

- Icon buttons without visible text → require `aria-label`
- Icons alongside text → `aria-hidden="true"` (decorative)
- `.hds-sr-only` utility class (uses `visually-hidden` mixin)
- Focus rings use `--hds-palette-link-text` with fallback to NASA Blue
- HDS wayfinding rule: red = navigate away, blue = on-page action

---

## Forms

- Light backgrounds only (stable palettes)
- Dark palette form elements are TODO
- Lean on USWDS defaults with HDS color overrides

---

## Print Styles

`@media print` in `_hds-custom-styles.scss` forces:
- White background, black text (all palettes)
- Hidden: nav, footer, banner, icon buttons
- Visible: link URLs after link text

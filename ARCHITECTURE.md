## HDS Core Architecture Decisions (as of 2026-03-08)

### Package
- `@nasa/hds-core` — CMS-agnostic Sass package on top of `@uswds/uswds ^3.3.0`
- Vanilla JS (no jQuery), SCSS source with CSS output
- Gulp + @uswds/compile for build, gulp-svg-sprite for icons
- No current consumers — still in draft/develop branch

### File Structure
```
src/scss/
  styles.scss                         → Entry point (9 forwards in order)
  _hds-tokens.scss                    → Brand hex vars, Sass maps, CSS custom props on :root
  _hds-uswds-theme.scss               → @use "uswds-core" with (...) — UNCHANGED
  _hds-uswds-theme-custom-styles.scss → USWDS component overrides
  _hds-mixins.scss                    → Typography, button, a11y mixins
  _hds-base.scss                      → HTML element styles with palette var fallbacks
  _hds-palettes.scss                  → 6 palette definitions (3 stable, 3 experimental)
  _hds-palette-elements.scss          → Palette-aware element + icon button component styles
  _hds-print.scss                     → Force light colors on print
```

### Palette System
- 6 palettes: **white** (default), **light**, **midtone**, **dark** (exp), **blue** (exp), **black** (exp)
- Apply via `class="hds-palette-dark"` or `data-hds-palette="dark"`
- 19 CSS custom properties per palette (simplified from ~30)
- `$hds-enable-experimental-palettes: false !default` — opt-in for palettes 4-6
- `$hds-enable-auto-dark-mode: false !default` — permanently opt-in, requires experimental flag
- USWDS utility classes (`!important`) override palette vars
- No palette attribute = normal USWDS behavior

### Palette Variables
**Universal (never varies):** `btn-primary-bg` (NASA Red), `btn-filled-text` (White), `checkbox-fill` (NASA Blue), `social-fill` (Carbon 60), `social-icon` (Carbon 05)

**Per-palette:** `bg`, `heading`, `text`, `link-text`, `link-underline`, `muted` (labels/metadata/captions), `border`, `btn-secondary-bg`, `checkbox-text`, `checkbox-stroke`, `utility-fill`, `utility-stroke`, `utility-icon`, `icon`

### Icons
- Flat directory, lowercase-hyphenated names with category prefix: `arrow-chevron-right.svg`, `file-archive.svg`, `tag-video.svg`
- System/arrow/file/eye/interactive icons: single-color `currentColor`, circles stripped (CSS provides container)
- Tag icons: keep built-in circle, `currentColor` on circle, white inner art
- Arrow/Circle-* files: kept as-is with embedded circles for devs who want them
- `viewBox="-2 -2 24 24"` is correct (20px art centered in 24px frame)
- Sprite generated by gulp-svg-sprite → `dist/assets/img/hds-sprite.svg`
- CSS references individual SVG files (mask-image can't use sprites); HTML uses sprite via `<use href>`
- `Primary-Arrow.svg` deleted — CSS `::after` generates it

### Icon Buttons (`.hds-btn-icon`)
- `.hds-btn-icon` = interactive circle container (needs `aria-label` when standalone)
- `.hds-icon` = SVG glyph inside (always `aria-hidden="true" focusable="false"`)
- `.hds-glyph` = bare inline icon without container
- Roles: `--cta` (red/navigate), `--secondary` (blue filled/on-page), `--outline` (blue border), `--utility` (neutral/adaptive), `--social` (gray), `--interactive` (small tooltip triggers)
- Sizes: `--sm` (inline with text), `--lg` (hero)

### Primary Arrow Button (`.hds-btn--primary`)
- Text + CSS `::after` red circle with white arrow
- Arrow direction auto-swaps: right (internal) → diagonal (external via `.usa-link--external`)
- Data URI SVGs for arrows (hardcoded white is correct per spec)

### External Links
- Uses USWDS's `.usa-link--external` class (manual, explicit)
- HDS overrides USWDS launch icon → HDS `arrow-line-diagonal.svg` via mask-image
- SR text provided by USWDS CSS (`::before` with sr-only)
- Optional JS auto-detect module (`hds-external-links.js`) — separate, not auto-loaded
- `.hds-link--internal` escape hatch

### Accessibility
- Icon buttons without text → require `aria-label`
- Icons alongside text → `aria-hidden="true"` (decorative)
- `.hds-sr-only` utility class (uses `visually-hidden` mixin)
- Focus rings use `--hds-palette-link-text` with fallback
- HDS wayfinding rule enforced by component roles (red=navigate, blue=on-page)

### Forms
- Light backgrounds only for now
- Dark palette forms are TODO
- Lean on USWDS defaults

### Print
- `@media print` forces all palettes to white bg, black text
- Hides nav, footer, banner, icon buttons
- Shows link URLs after link text
